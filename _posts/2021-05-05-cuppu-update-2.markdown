## Cuppu86 development resumed
Now that I don't have all the academic pressure that was there up until a few days ago, I've resumed the development of Cuppu86! [samarthkm][1] is free now as well so he's back as well. We've had a bit of trouble trying to split the work amongst ourselves without any hickups, but we hadn't had any success. But we're trying to do things differently now. Initially, the terminal client for [cuppu86][2] was being developed in the same place as the other components. This caused weird issues with git, wherein we couldn't maintain the git repo very well. So we've chosen to split the client into its own separate repo which can be found [here][3].

According to this new arrangement, [samarthkm][1] will be in charge of developing the terminal client. He's also taken the responsibility of testing the code as and when he can. This also means that I'm no longer going to be involved in the testing process, and will solely be focussing on implementing the complete 8086 instruction set.

I want to try and complete this project by the end of May. I've never worked with a deadline before so I'm excited for this one 😂.
